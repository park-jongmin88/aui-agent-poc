# validate - 템플릿 규격 검증

## 개념
run.py가 어떻게 만들어졌든(에이전트 생성, 기존 코드 변환, 숙련자 직접 작성)
9-섹션 표준만 지키면 train으로 진행 가능하다. validate는 그 관문이다.

## 절차
1. 대상 폴더를 확인한다. 지정이 없으면 workspace/models/ 하위 폴더 목록을 보여주고 선택받는다.
2. 대상 run.py 와 workspace/templates/run.py 를 모두 읽는다.
3. 아래 9개 섹션 주석이 모두 존재하고 순서가 맞는지 비교한다:
   1) 임포트  2) MLflow 연동  3) 데이터 준비  4) 모델 준비  5) 트레이닝 펑션
   6) 인풋 샘플  7) MLflow 모델 로깅  8) config.json 정의  9) 런 스타트
4. 채워지지 않은 `# TODO:` 항목이 남아있는지 확인한다.
5. 결과를 [통과 / 항목별 미비 사항] 형식으로 보고한다.

## 주의
- 검증만 수행하고 파일을 수정하지 않는다. 수정은 사용자 동의 후에만.
